# 审查 Agent Prompt 模板

使用方式：每轮审查启动 N 个并行 agent，每人负责一章。修改 `[章节名]` 和 `[素材列表]` 后发送。

---

## 模板一：内容完整性审查（第一轮）

```
审查 `[.tex文件路径]`，判断内容是否完整。

**对比素材**：[列出该章相关的素材文件路径，如教材第X章PDF、习题册相关页码]

**审查项**：
1. 逐知识点与素材对比，是否有遗漏
2. 例题数量是否充足（每个知识点至少1题），类型是否覆盖考点
3. 公式、定理描述是否与教材一致，无笔误
4. 是否有该讲但没讲的重要题型

**输出格式**：
- 发现问题：行号 + 具体问题 + 建议补充的内容
- 全部完整：说"内容完整，无遗漏"
- 只输出审查报告，不修改文件
```

---

## 模板二：范围合规审查（第二轮）

```
审查 `[.tex文件路径]`，判断内容是否超出考试范围。

**考试范围**：[数三/数一/某校期末考试 等]
**参考教材**：[教材名]
**习题来源**：[660题/课后题/考卷 等]

**审查项**：
1. 是否有超纲知识点（如数三不考的方向导数、欧拉方程等，标注行号和原因）
2. 是否有教材和习题中都没出现过的知识点或技巧
3. 例题难度是否与习题匹配（太难或太简单都要指出）
4. 全文中是否残留"660""武忠祥"等具体机构名/教师名/题册名

**输出格式**：
- 发现问题：行号-问题-建议（删除/修改/替换）
- 全部合规：说"全部通过，无超纲内容"
- 只输出审查报告，不修改文件
```

---

## 模板三：结构体系审查（第三轮）

```
审查 `[.tex文件路径]`，判断文件结构是否成体系。

**期望的知识逻辑**：[如：概念→直角坐标→极坐标→换序→对称→应用]

**审查项**：
1. 章节排列是否按逻辑递进（不是拼凑），如果不合理，建议正确顺序
2. 是否存在独立附录章节（如"综合知识补充""常见误区集中警示""计算模板总结""典型综合题解析"等标题）
3. 补充/进阶内容是否已融入正文对应小节
4. 是否存在内容重复（同一知识点在两个地方讲）
5. 副标题、studybox 中是否列举了不在考试范围内的内容
6. 文档是否给人"先讲完所有知识点，然后丢一堆补充"的拼凑感

**输出格式**：
- 发现问题：行号-问题-建议修改
- 全部通过：说"体系完整，通过"
- 只输出审查报告，不修改文件
```

---

## 模板四：综合最终审查

```
审查 `[.tex文件路径]`，做最终全面检查。

**要求**：
1. 数三范围（或指定范围）：无超纲内容
2. 无独立附录章节（综合知识补充等已融入正文）
3. 体系完整（逻辑递进，非拼凑）
4. 无机构名/教师名残留
5. 无超纲例题
6. 无内容重复

**输出格式**：以表格列出每项通过/不通过，不通过项标注行号和修改建议。只审查不修改。
```
