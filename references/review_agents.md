# 审查 Agent Prompt 模板

使用方式：审查阶段启动 N 个并行 agent，每人负责一章。修改 `[章节名]` 和 `[素材列表]` 后发送。

---

## 模板一：合并审查（一次覆盖三维度）

```
审查 `[.tex文件路径]`，从以下三个维度一次性检查，输出综合问题清单。

**考试范围**：[数三/数一/某校期末考试 等]
**参考教材**：[教材名]
**对比素材**：[列出该章相关的素材文件路径，如教材第X章PDF、习题册相关页码]
**习题来源**：[660题/课后题/考卷 等]
**期望的知识逻辑**：[如：概念→直角坐标→极坐标→换序→对称→应用]

---

### 维度一：内容完整性
1. 逐知识点与素材对比，是否有遗漏
2. 例题数量是否充足（每个知识点至少1题），类型是否覆盖考点
3. 公式、定理描述是否与教材一致，无笔误
4. 是否有该讲但没讲的重要题型

### 维度二：范围合规
1. 是否有超纲知识点（标注行号和原因）
2. 是否有教材和习题中都没出现过的知识点或技巧
3. 例题难度是否与习题匹配（太难或太简单都要指出）
4. 全文中是否残留具体机构名/教师名/题册名（如"660""武忠祥"等）

### 维度三：结构体系
1. 章节排列是否按逻辑递进（不是拼凑），如果不合理，建议正确顺序
2. 是否存在独立附录章节（如"综合知识补充""常见误区集中警示""计算模板总结""典型综合题解析"等标题）
3. 补充/进阶内容是否已融入正文对应小节
4. 是否存在内容重复（同一知识点在两个地方讲）
5. 副标题、studybox 中是否列举了不在考试范围内的内容
6. 文档是否给人"先讲完所有知识点，然后丢一堆补充"的拼凑感

---

**输出格式**：
按维度分组输出：
- **完整性**：行号 + 具体问题 + 建议补充的内容（无问题则写"通过"）
- **范围**：行号 + 问题 + 建议（删除/修改/替换）（无问题则写"通过"）
- **结构**：行号 + 问题 + 建议修改（无问题则写"通过"）

只输出审查报告，不修改文件。
```

---

## 模板二：最终核验

```
核验 `[.tex文件路径]`，对修改后的讲义做最终关门检查。

**用户原始需求**：[粘贴前置确认的全部结论：考试范围、素材来源、习题要求、章节规划、视觉上限等]
**上一轮问题清单**：[粘贴该章的问题列表]
**对比素材**：[该章对应的 chXX_parsed.md 路径]

**检查项**：
1. 上一轮问题是否每一条都已修复（逐条核对）
2. 修改过程是否引入新问题，对照用户原始需求检查：
   - 内容是否仍然覆盖素材中所有要求的知识点/例题
   - 是否引入了需求范围外的内容
   - 结构是否仍满足要求（无拼凑附录、逻辑递进等）
   - 公式、符号是否有笔误
   - 敏感词是否已清除
3. PDF 能否正常编译，无报错

**输出格式**：
- 逐条标注旧问题：已修复 / 未修复（说明现状）
- 有新问题：行号 + 描述
- 全部通过：说"核验通过"

只输出核验报告，不修改文件。
```
